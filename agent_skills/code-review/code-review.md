# Code Review Skill

在审查代码时，请遵循以下步骤：

## 审查清单

1. **正确性 (Correctness)**：代码是否实现了预期的功能？
2. **边缘情况 (Edge cases)**：是否处理了错误条件和异常输入？
3. **风格 (Style)**：是否符合项目编码规范和约定？
4. **性能 (Performance)**：是否存在明显的低效代码或资源泄露？

## 如何提供反馈

- 具体说明需要更改的地方。
- 解释“为什么”要改，而不仅仅是“改什么”。
- 尽可能提供改进的代码示例或替代方案。
